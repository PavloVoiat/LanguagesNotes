1. 📌 **Тема**: Циклы (for, while): автоматизация повторяющихся действий

2. 🔍 **Разбор концепции**: 
На прошлом занятии мы научили программу принимать решения с помощью условий. Но что, если нам нужно выполнить одно и то же действие 100 раз? Писать 100 строк кода — плохая практика. Здесь на помощь приходят *циклы*. *Цикл* — это управляющая конструкция, которая повторяет блок кода, пока заданное условие остается истинным. Самый популярный — это цикл `for`. Он состоит из трех частей: инициализация счетчика, условие выхода и шаг изменения счетчика. Представь это как конвейер на заводе, который работает, пока не кончатся детали в ящике.

3. 💡 **Зачем это нужно**?: 
В реальных проектах ты постоянно будешь работать с массивами данных. Например, пришел список из 50 пользователей с сервера, и тебе нужно вывести имя каждого из них на страницу. Вместо дублирования кода ты используешь цикл, который проходит по списку и делает отрисовку для каждого элемента автоматически.

4. 💻 **Живой пример**:

```JavaScript
// Пример цикла for: выводим числа от 1 до 5
for (let i = 1; i <= 5; i++) {
  console.log("Номер итерации: " + i);
}

// Пример цикла while: работаем, пока условие верно
let count = 0;
while (count < 3) {
  console.log("Счетчик: " + count);
  count++;
}
```

5. ⚠️ **Типичные ошибки**:
- <u>Бесконечный цикл</u>: если ты забудешь изменить значение счетчика (например, не напишешь `i++`), условие всегда будет истинным, и браузер зависнет, так как будет пытаться выполнить код вечно.
- <u>Ошибка "off-by-one"</u>: когда программист путает `<` и `<=`, из-за чего цикл выполняется на один раз больше или меньше нужного. <u>Всегда тщательно проверяй границы диапазона</u>.

5. 🛠 **Практический челлендж**:
Напиши цикл `for`, который выводит в консоль только четные числа от 0 до 10 включительно. 
Подсказка: используй оператор остатка от деления `%` (например, `i % 2 === 0` проверяет, является ли число четным).

✅ **Ключевой вывод**: 
Циклы позволяют нам не писать рутинный код вручную. Они — основа обработки любых списков данных. Главное правило: всегда имей четкое условие выхода из цикла, чтобы не допустить зависания программы.
