1. 📌 **Тема**: Хранение данных на клиенте: LocalStorage, SessionStorage и Cookies. 

2. 🔍 **Разбор концепции**: 
В веб-разработке нам часто нужно, чтобы данные "жили" дольше, чем одна сессия страницы (например, тема оформления сайта или корзина товаров). Для этого браузер предоставляет механизмы *Web Storage API*. 
*LocalStorage* — позволяет хранить данные без срока годности, они остаются в браузере даже после закрытия вкладки или выключения компьютера. 
*SessionStorage* — хранит данные только на время текущей сессии (пока открыта вкладка). 
*Cookies* — это небольшие текстовые файлы, которые сервер отправляет браузеру. Они часто используются для *аутентификации*, так как браузер автоматически передает их при каждом запросе к серверу.

3. 💡 **Зачем это нужно?**: 
Представь, что пользователь выбрал "Темную тему". Ты не хочешь, чтобы при каждом обновлении страницы сайт снова становился светлым. Ты сохраняешь выбор пользователя в *LocalStorage*. А если пользователь залогинился, то токен доступа (JWT) часто сохраняют в *Cookies* с флагом *HttpOnly* для защиты от XSS-атак.

4. 💻 **Живой пример**:

```JavaScript
// Сохранение данных в LocalStorage
localStorage.setItem('theme', 'dark');

// Получение данных
const currentTheme = localStorage.getItem('theme');
console.log(currentTheme); // "dark"

// Удаление данных
localStorage.removeItem('theme');

// Работа с объектами (необходимо сериализовать в JSON)
const user = { id: 1, name: 'Alex' };
localStorage.setItem('user', JSON.stringify(user));

const savedUser = JSON.parse(localStorage.getItem('user'));
console.log(savedUser.name); // "Alex"
```

5. ⚠️ **Типичные ошибки**:
- <u>Попытка сохранить объект напрямую</u>: <u>LocalStorage хранит только строки</u>. Если записать объект как есть, ты получишь `"[object Object]"`. Всегда используй `JSON.stringify()` для записи и `JSON.parse()` для чтения.
- <u>Хранение паролей или очень чувствительных данных в LocalStorage</u>: <u>LocalStorage доступен любому JavaScript коду</u> на странице, поэтому это небезопасно. 
- <u>Забывать про лимиты</u>: <u>Объем хранилища ограничен</u> (обычно около 5MB). Не стоит хранить там огромные массивы данных.

6. 🛠 **Практический челлендж**:
Напиши функцию `saveUserData(data)`, которая принимает объект с именем пользователя и его предпочтениями, преобразует его в строку и сохраняет в *LocalStorage*. Затем напиши функцию `getUserData()`, которая достает эти данные и возвращает их обратно в виде объекта. Вызови их, чтобы проверить работоспособность.
