1. 📌 **Тема**: Асинхронность и событийный цикл (Event Loop). Введение в Promise.

2. 🔍 **Разбор концепции**:
JavaScript — это *однопоточный язык*. Это значит, что он может выполнять только одну операцию за раз. Но как же тогда он обрабатывает запросы к серверу или таймеры, не «зависая» наглухо? Здесь на сцену выходит *Event Loop* (цикл событий). 
Когда мы вызываем асинхронную операцию, браузер делегирует её выполнение своей Web API. Как только задача готова, её результат попадает в *очередь микрозадач* или *очередь макрозадач*. *Event Loop* постоянно проверяет: пуст ли основной стек вызовов? Если да, он берет задачу из очереди и переносит её в основной поток. *Promise* — это объект, который представляет собой «обещание» вернуть результат операции в будущем (успешно или с ошибкой).

3. 💡 **Зачем это нужно?**:
Без асинхронности ваш сайт «замерзал» бы каждый раз при загрузке данных из базы данных или при ожидании ответа от API. Пользователь не смог бы нажать ни на одну кнопку, пока скрипт ждет ответа от сервера. *Promise* позволяет писать код, который читается почти как синхронный, избегая «ада обратных вызовов» (callback hell).

4. 💻 **Живой пример**:

```JavaScript
// Создаем промис, имитирующий запрос к серверу
const fetchData = new Promise((resolve, reject) => {
  const success = true;
  setTimeout(() => {
    if (success) {
      resolve("Данные получены!");
    } else {
      reject("Ошибка сервера");
    }
  }, 2000);
});

console.log("Запрос отправлен...");

fetchData
  .then((data) => console.log(data)) // Выполнится при успехе
  .catch((error) => console.error(error)) // Выполнится при ошибке
  .finally(() => console.log("Операция завершена.")); // Выполнится в любом случае
```

5. ⚠️ **Типичные ошибки**:
- <u>Забывать обрабатывать ошибку</u> через метод `.catch()`. Если промис упадет с ошибкой, а обработчика нет, вы получите "Unhandled Promise Rejection".
- <u>Попытка получить значение промиса в переменную вне </u>`.then()`. Промис — это объект, он не возвращает значение мгновенно. Нельзя просто написать `const result = fetchData`, нужно всегда работать внутри цепочки методов или использовать `async/await` (о них мы поговорим позже).

6. 🛠 **Практический челлендж**:
Создайте промис `waitOneSecond`, который через 1 секунду вызывает `resolve("Время вышло!")`. Вызовите этот промис и выведите результат в консоль через `.then()`. Бонус: добавьте `console.log` до вызова промиса и после него, чтобы увидеть, в каком порядке они появятся в консоли.

✅ **Ключевой вывод**: JavaScript не блокирует выполнение кода благодаря *Event Loop*. *Promise* — это удобный инструмент для управления асинхронными операциями, позволяющий элегантно обрабатывать успешные ответы и ошибки, не останавливая основной поток выполнения программы.
