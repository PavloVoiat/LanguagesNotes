1. 📌 **Тема**: Переменные и способы их объявления (let, const, var)

2. 🔍 **Разбор концепции**: 
Представь, что оперативная память компьютера — это огромный склад с ячейками. Когда мы объявляем переменную, мы помечаем одну из ячеек нужным нам именем, чтобы позже найти там значение. В JavaScript существует *область видимости* — это границы, внутри которых переменная "видна" программе. 
- `var` — старый способ, который игнорирует границы блоков кода (фигурные скобки) и часто ведет к ошибкам.
- `let` — позволяет создавать переменные, значение которых можно изменять.
- `const` — от слова *constant*, создает переменную, которую нельзя переназначить.

3. 💡 **Зачем это нужно?**: 
В реальных проектах мы используем переменные, чтобы хранить состояние приложения: данные пользователя, настройки темы, счетчики товаров в корзине. Правильный выбор инструмента защищает твой код от случайных перезаписей данных, которые могут "сломать" логику приложения в самый неподходящий момент.

4. 💻 **Живой пример**:

```JavaScript
// Используем const для того, что не должно меняться
const userName = "Алексей"; 

// Используем let для счетчиков или данных, которые меняются
let userScore = 0;
userScore = 10;

console.log(userName); // Алексей
console.log(userScore); // 10
```

5. ⚠️ **Типичные ошибки**:
Новички часто используют `var` по привычке, что создает дыры в безопасности кода. Также важно помнить: если ты объявил переменную через `const`, ты не можешь присвоить ей новое значение — <u>программа выдаст ошибку</u>. И помни про регистр: `userName` и `username` — это <u>две разные переменные</u>!

6. 🛠 **Практический челлендж**:
Создай две переменные: одну для хранения твоего имени (которое не меняется), а вторую — для количества прочитанных тобой книг (которое будет расти). Выведи их в консоль. Попробуй изменить значение первой переменной и посмотри, что произойдет в консоли браузера.

✅ **Ключевой вывод**: Всегда используй `const` по умолчанию, если значение не должно меняться, и `let`, если оно будет обновляться. Избегай использования `var` в современном коде.
