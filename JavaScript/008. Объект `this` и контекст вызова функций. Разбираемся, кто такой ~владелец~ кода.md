1. 📌 **Тема**: Объект `this` и контекст вызова функций. Разбираемся, кто такой "владелец" кода.

2. 🔍 **Разбор концепции**: 
После того как мы изучили замыкания, пора понять, как функции узнают, с какими данными работать. В JavaScript существует *контекст выполнения*, а ключевое слово `this` — это ссылка на объект, который вызывает функцию в данный момент. 
Важно помнить: `this` не зависит от места написания функции, он зависит от того, <u>как именно</u> мы вызываем функцию. 
- Если метод вызван через точку (например, `user.sayHi()`), то this — это объект `user`.
- Если функция вызвана просто как `func()`, то в строгом режиме this будет `undefined`.
- В стрелочных функциях `this` особенный: у них нет своего контекста, они "заимствуют" его из внешней области видимости.

3. 💡 **Зачем это нужно?**: 
В реальных проектах мы постоянно работаем с объектами. Например, если у тебя есть компонент кнопки на сайте, при клике тебе нужно обратиться к данным именно этой кнопки (её цвету, тексту или ID). `this` позволяет писать универсальный код, который понимает, к какому объекту он привязан в текущий момент.

4. 💻 **Живой пример**:

```JavaScript
const user = {
  name: 'Алексей',
  sayHello() {
    console.log(`Привет, меня зовут ${this.name}`);
  }
};

user.sayHello(); // Привет, меня зовут Алексей

const anotherUser = { name: 'Мария' };
anotherUser.sayHello = user.sayHello;

anotherUser.sayHello(); // Привет, меня зовут Мария
```

5. ⚠️ **Типичные ошибки**:
- <u>Потеря контекста</u>: когда метод объекта передается как колбэк (например, в `setTimeout` или обработчик события), он теряет связь со своим объектом.
- <u>Использование стрелочных функций как методов объекта</u>: стрелочная функция возьмет `this` из глобальной области (обычно `window` или `undefined`), а не из объекта, что приведет к ошибке `undefined`.
- <u>Вызов без </u>`new`<u> для функций-конструкторов</u>: <u>всегда</u> проверяйте, как вызывается ваша функция, иначе `this` будет указывать не туда.

6. 🛠 **Практический челлендж**:
Создай объект `calculator` с методами `read`, `add` и `subtract`. Метод `read` должен запрашивать через `prompt` два числа и сохранять их как свойства объекта. Методы `add` и `subtract` должны использовать эти свойства через `this` и возвращать результат суммы или разности.

✅ **Ключевой вывод**: `this` — это динамическая ссылка. Она определяется только в момент вызова функции. Понимание того, кто является "хозяином" функции, спасает от 90% багов, связанных с недоступностью данных внутри методов.
