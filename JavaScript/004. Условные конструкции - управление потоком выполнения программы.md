1. 📌 **Тема**: Условные конструкции: управление потоком выполнения программы

2. 🔍 **Разбор концепции**:
После того как мы научились работать с типами данных и операторами, пришло время научить программу принимать *решения*. *Условные конструкции* — это механизмы, которые позволяют коду выполнять разные действия в зависимости от того, истинно или ложно условие. По сути, это *ветвление* вашего алгоритма. Мы используем логику "если это правда — делай А, иначе — делай Б". В JavaScript это реализуется через `if`, `else if`, `else`, а также через *тернарный оператор* для простых случаев.

3. 💡 **Зачем это нужно?**:
В реальной разработке это основа всего. Представь интернет-магазин: если у пользователя есть <u>подписка</u>, мы показываем ему скидку, если <u>баланс</u> выше цены товара — разрешаем покупку, иначе — просим пополнить счет. Без условий программа была бы линейной и бесполезной.

4. 💻 **Живой пример**:

```JavaScript
const userBalance = 50;
const itemPrice = 100;

// Классическая конструкция
if (userBalance >= itemPrice) {
  console.log("Покупка разрешена");
} else if (userBalance > 0) {
  console.log("Недостаточно средств, но вы близки к цели!");
} else {
  console.log("Пополните баланс");
}

// Тернарный оператор (для коротких условий)
const message = userBalance >= itemPrice ? "Успех!" : "Ошибка!";
console.log(message);
```

5. ⚠️ **Типичные ошибки**:
- <u>Забытые фигурные скобки</u>: если тело условия состоит из одной строки, скобки можно не ставить, но это плохая практика, ведущая к багам при расширении кода.
- <u>Путаница между присваиванием и сравнением</u>: никогда не пишите `if (a = 5)`, это всегда будет `true`, так как вы присваиваете значение. Используйте `===` для проверки.
- <u>Избыточность</u>: новички часто пишут `if (isLogged === true)`, хотя достаточно просто `if (isLogged)`.

6. 🛠 **Практический челлендж**:
Напиши скрипт, который принимает переменную `age` (число). 
- Если `age` меньше 18, выведи "Доступ запрещен".
- Если от 18 до 60 (включительно) — "Добро пожаловать".
- Если больше 60 — "Приходите за пенсионной скидкой".
Попробуй решить эту задачу, используя `if/else`, а затем перепиши самую простую часть через тернарный оператор.

✅ **Ключевой вывод**: Условные конструкции позволяют коду быть *динамичным*. Мы используем `if/else` для сложных проверок и тернарный оператор для лаконичного выбора между двумя значениями. Помни: всегда проверяй тип и значение с помощью `===`, чтобы избежать сюрпризов при приведении типов, о которых мы говорили на прошлом уроке.
