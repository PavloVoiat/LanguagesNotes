1. 📌 **Тема**: Продвинутая работа с массивами: методы и итерация (функциональный подход)

2. 🔍 **Разбор концепции**: 
В современной разработке мы стараемся избегать классических циклов `for` в пользу *методов высшего порядка*. Это функции, которые принимают другую функцию в качестве аргумента (callback). Основная идея заключается в *декларативном подходе*: мы описываем «что» хотим получить от массива, а не «как» именно перебирать элементы. Ключевые методы — `map`, `filter`, `reduce` и `find`. Они позволяют писать чистый и предсказуемый код, избегая побочных эффектов.

3. 💡 **Зачем это нужно?**: 
Представь, что у тебя есть список товаров из базы данных, и тебе нужно получить только те, которые стоят дешевле 1000 рублей, а затем создать массив их названий для отображения в UI. Использование `filter` и `map` делает код лаконичным, а главное — позволяет избежать мутации исходных данных, что критически важно в архитектуре крупных приложений.

4. 💻 **Живой пример**:

```JavaScript
const products = [
  { id: 1, name: 'Книга', price: 500 },
  { id: 2, name: 'Ноутбук', price: 50000 },
  { id: 3, name: 'Ручка', price: 50 }
];

// Фильтруем дешевые товары и трансформируем их в массив имен
const cheapProductNames = products
  .filter(product => product.price < 1000)
  .map(product => product.name);

console.log(cheapProductNames); // ["Книга", "Ручка"]

// Считаем общую стоимость всех товаров через reduce
const total = products.reduce((sum, item) => sum + item.price, 0);
console.log(total); // 50550
```

5. ⚠️ **Типичные ошибки**:
- <u>Забывать возвращать значение</u>: в методах `map` или `filter` обязательно нужен `return` (или использование стрелочной функции без фигурных скобок, тогда `return` подразумевается).
- <u>Мутация исходного массива</u>: использование методов типа `sort` или `reverse`, которые меняют исходный массив, вместо создания копий.
- <u>Неправильное использование </u>`reduce`: новички часто забывают передать <u>начальное значение</u> (второй аргумент 0), что может привести к ошибке, если массив окажется пустым.

6. 🛠 **Практический челлендж**:
У тебя есть массив чисел: `[10, 25, 40, 5, 80]`. 
Твоя задача: 
- С помощью метода `filter` создай новый массив, где останутся только числа больше 20.
- С помощью метода `map` умножь каждое из оставшихся чисел на 2.
- Выведи результат в консоль.
Попробуй написать решение в одну строку, используя цепочку методов (метод `chaining`).

✅ **Ключевой вывод**: Использование функциональных методов массивов превращает сложную логику обработки данных в *читаемый поток* операций. Это стандарт индустрии, который делает твой код профессиональным и защищенным от случайных ошибок при переборе.
