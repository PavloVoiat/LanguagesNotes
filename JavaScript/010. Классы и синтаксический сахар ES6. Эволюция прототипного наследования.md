1. 📌 *Тема*: Классы и синтаксический сахар ES6. Эволюция прототипного наследования.

2. 🔍 **Разбор концепции**: 
После того как мы разобрались с тем, как прототипы работают «под капотом», пришло время познакомиться с более удобным инструментом. *Классы* в JavaScript — это не отдельный механизм, а *синтаксический сахар*. Это означает, что `class` в JS делает всё то же самое, что и прототипы, которые мы изучили в прошлом уроке, но записывается это гораздо чище и понятнее для разработчика, пришедшего из других языков (Java, C#). Главные компоненты: *constructor* (метод для инициализации свойств) и *методы класса* (функции, которые автоматически попадают в прототип).

3. 💡 **Зачем это нужно?**: 
В реальных проектах ты будешь создавать сотни сущностей: пользователей, товары в корзине, компоненты интерфейса. Использовать `Object.create` или менять `prototype` вручную при масштабировании становится утомительно. Классы позволяют структурировать код, делая его читаемым, предсказуемым и удобным для командной работы.

4. 💻 **Живой пример**:

```JavaScript
class User {
  constructor(name, role) {
    this.name = name;
    this.role = role;
  }

  sayHello() {
    console.log(`Привет, я ${this.name}, моя роль: ${this.role}`);
  }
}

// Наследование
class Admin extends User {
  constructor(name) {
    super(name, 'Администратор');
  }

  deleteUser(user) {
    console.log(`Пользователь ${user} удален`);
  }
}

const admin = new Admin('Иван');
admin.sayHello();
admin.deleteUser('Петр');
```

5. ⚠️ **Типичные ошибки**:
- <u>Забывать вызывать </u>`super()` в конструкторе дочернего класса. Если ты наследуешься, `super()` обязан быть первой строкой в конструкторе, иначе JS выбросит ошибку.
- <u>Попытка использовать стрелочные функции</u> как методы класса, если нужно обращение к `this` в динамическом контексте (хотя это лечится публичными полями классов, о чем позже).
- <u>Смешивание парадигм</u>: иногда новички пытаются добавлять методы напрямую в экземпляр (`this.method = ...`), вместо того чтобы определять их как методы класса. Это потребляет больше памяти, так как каждый объект получает свою копию функции.

6. 🛠 **Практический челлендж**:
Создай класс `Shape` (Фигура) с конструктором, принимающим `color`. Добавь метод `getColor`. Затем создай класс `Circle`, который наследуется от `Shape`, принимает в конструктор `color` и `radius`. Реализуй в `Circle` метод `getArea`, который возвращает площадь круга (`PI * radius * radius`). Создай экземпляр круга и выведи его цвет и площадь в консоль.

✅ **Ключевой вывод**: <u>Классы — это удобная «обертка» над прототипами</u>. Они помогают писать чистый код, соблюдая принципы ООП, но важно помнить, что под ними всё ещё скрывается старая добрая прототипная цепочка.
