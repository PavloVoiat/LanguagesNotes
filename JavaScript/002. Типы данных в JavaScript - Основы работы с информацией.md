1. 📌 **Тема**: Типы данных в JavaScript: Основы работы с информацией

2. 🔍 **Разбор концепции**: 
В прошлом уроке мы научились создавать "коробки" (переменные). Теперь разберемся, что именно мы можем в них положить. В JavaScript данные делятся на две большие категории. ***Примитивные типы*** — это базовые строительные блоки: *строки* (текст), *числа* (целые и дробные), *булевы значения* (истина или ложь), *null* (явное отсутствие значения) и *undefined* (значение не было присвоено). Важно помнить: ***примитивы неизменяемы***, то есть мы не можем изменить значение "на лету", мы можем только заменить его новым.

3. 💡 **Зачем это нужно?**: 
Представь, что ты пишешь приложение для интернет-магазина. Имя пользователя — это *строка*, цена товара — *число*, а статус "товар в наличии" — это *булево значение* (true или false). Без понимания типов данных ты не сможешь правильно производить вычисления или проверять условия.

4. 💻 **Живой пример**:

```JavaScript
// Примитивные типы данных
const name = "Алексей"; // string
const age = 25; // number
const isStudent = true; // boolean
const emptyValue = null; // null
let unknownValue; // undefined (автоматически присваивается, если значение не задано)

console.log(typeof age); // "number"
console.log(typeof isStudent); // "boolean"
```

5. ⚠️ **Типичные ошибки**:
Первая ошибка — это <u>сложение строки и числа</u>. Если ты сделаешь `"5" + 5`, JavaScript превратит число в строку, и результат будет `"55"`, а не `10`. Вторая частая ложь — считать, что `null` и `undefined` это одно и то же. Запомни: <u>null — это "пусто", а undefined — это "ничего не задали"</u>. И не забывай про <u>регистрозависимость</u>: значение `true` и `True` — это разные вещи для интерпретатора.

6. 🛠 **Практический челлендж**:
Создай 4 переменные с разными типами данных (строка, число, булево значение и undefined). Выведи в консоль тип каждой из них с помощью оператора `typeof`. Попробуй сложить переменную-строку и переменную-число, посмотри, что получится в консоли, и объясни себе, почему результат именно такой.

✅ **Ключевой вывод**: Мы узнали, что данные имеют свой *тип*, который определяет, что мы можем делать с этим значением. Понимание типов — это база для предотвращения багов при выполнении математических операций или логических проверок.
