1. 📌 **Тема**: Async/Await: Элегантная асинхронность и обработка ошибок

2. 🔍 **Разбор концепции**: 
В прошлом уроке мы познакомились с *Promise*. Они решили проблему «адской пирамиды» (callback hell), но цепочки из `.then().then()` все еще могут выглядеть громоздко. *Async/await* — это синтаксический сахар, построенный поверх промисов, который делает асинхронный код визуально похожим на синхронный. Когда мы помечаем функцию как `async`, она автоматически возвращает промис. Ключевое слово `await` заставляет JavaScript <u>приостановить выполнение функции</u> до тех пор, пока промис не перейдет в состояние *resolved* (исполнено), после чего вернет результат. Если промис перейдет в *rejected*, выбросится исключение, которое можно поймать привычным блоком `try...catch`.

3. 💡 **Зачем это нужно?**: 
В реальных проектах мы постоянно работаем с сервером (API). Представь, что тебе нужно сначала авторизовать пользователя, затем получить его профиль, а потом список его заказов. С помощью *async/await* ты пишешь код линейно, что в разы проще читать, отлаживать и поддерживать по сравнению с вложенными цепочками методов.

4. 💻 **Живой пример**:

```JavaScript
// Функция для симуляции запроса к серверу
const fetchUserData = (id) => {
  return new Promise((resolve) => {
    setTimeout(() => resolve({ id, name: "Иван" }), 1000);
  });
};

// Использование async/await
async function showUser() {
  try {
    console.log("Загрузка...");
    const user = await fetchUserData(1);
    console.log("Пользователь получен:", user.name);
  } catch (error) {
    console.error("Ошибка при загрузке:", error);
  } finally {
    console.log("Запрос завершен");
  }
}

showUser();
```

5. ⚠️ **Типичные ошибки**:
- <u>Забывать блок try...catch</u>: Если промис завершится с ошибкой (reject), а обертки нет, программа "упадет" с необработанной ошибкой.
- <u>Использование await вне async-функции</u>: В старых средах это приведет к синтаксической ошибке (хотя современные браузеры поддерживают top-level await в модулях).
- <u>Последовательный запуск вместо параллельного</u>: <u>Никогда не делай </u>`await` один за другим, если запросы не зависят друг от друга. Используй `Promise.all`, чтобы запускать их одновременно и экономить время.

6. 🛠 **Практический челлендж**:
Напиши функцию `getPostData`, которая:
- Создает промис, который через 2 секунды возвращает объект `{ title: "Привет, мир" }`.
- Использует `async/await` для получения этого объекта.
- Выводит заголовок в консоль.
- Оберни всё в `try...catch`, чтобы обработать потенциальную ошибку.

✅ **Ключевой вывод**: *Async/await* превращает сложную цепочную логику в понятный, линейный код. Это стандарт индустрии, который делает работу с асинхронностью предсказуемой и безопасной благодаря использованию обычных блоков обработки ошибок.
