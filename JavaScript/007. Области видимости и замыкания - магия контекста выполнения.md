1. 📌 **Тема**: Области видимости и замыкания: магия контекста выполнения

2. 🔍 **Разбор концепции**:
После того как мы научились создавать функции, пришло время понять, где именно "живут" наши переменные. В JavaScript существует *область видимости* — это зона программы, где переменная доступна для использования. 
*Глобальная область видимости* доступна везде, а *локальная* ограничена фигурными скобками `{}` (блочная) или телом функции.
*Замыкание* — это способность функции "запоминать" окружение, в котором она была создана, даже если эта функция вызывается вне своего исходного контекста. Это как если бы функция сложила нужные ей переменные в свой "рюкзак" при рождении.

3. 💡 **Зачем это нужно?**:
В реальной разработке замыкания позволяют создавать *приватные данные*. Например, если вы пишете счетчик кликов, вы можете скрыть переменную счетчика внутри функции, чтобы из консоли браузера никто не мог изменить её значение напрямую. Это обеспечивает безопасность и инкапсуляцию логики вашего приложения.

4. 💻 **Живой пример**:

```JavaScript
function createCounter() {
  let count = 0; // Эта переменная "заперта" внутри
  
  return function() {
    count++; // Функция имеет доступ к переменной из внешней функции
    return count;
  };
}

const myCounter = createCounter();
console.log(myCounter()); // 1
console.log(myCounter()); // 2
```

5. ⚠️ **Типичные ошибки**:
- <u>Попытка доступа к локальной переменной снаружи функции</u>: вы получите ошибку `ReferenceError`.
- <u>Забытое ключевое слово</u>: если внутри функции вы присвоите значение переменной без `let`, `const` или `var`, она автоматически станет глобальной, что приведет к трудноуловимым багам. Всегда <u>объявляйте</u> переменные явно.

6. 🛠 **Практический челлендж**:
Напиши функцию `createGreeting(name)`, которая возвращает другую функцию. Внутренняя функция при вызове должна выводить в консоль строку: `"Привет, [name]!"`. 
Создай два разных приветствия: `sayHello = createGreeting('Алексей')` и `sayHi = createGreeting('Мария')`. Убедись, что они сохраняют свои имена независимо друг от друга.

✅ **Ключевой вывод**: <u>Область видимости</u> определяет доступность переменных, а <u>замыкания</u> позволяют функциям хранить состояние, что делает ваш код более предсказуемым и профессиональным.
