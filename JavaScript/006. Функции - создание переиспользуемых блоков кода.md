1. 📌 **Тема**: Функции: создание переиспользуемых блоков кода

2. 🔍 **Разбор концепции**: 
До этого момента мы писали код линейно. Но что, если нам нужно выполнить одно и то же действие 10 раз в разных частях программы? Копипастить код — плохая практика. Здесь на помощь приходят *функции*. 
*Функция* — это «черный ящик», который принимает на вход данные (*аргументы*), выполняет внутри себя заданный алгоритм и, если нужно, возвращает результат (*return*). Представь, что ты пишешь инструкцию один раз, а потом просто вызываешь её по имени, когда она понадобится. Это основа структуры любого серьезного приложения.

3. 💡 **Зачем это нужно?**: 
В реальной разработке мы не пишем весь код в одной «портянке». Мы разбиваем логику на маленькие, понятные части. Например, одна функция считает скидку в интернет-магазине, другая — форматирует дату, третья — отправляет запрос на сервер. Это делает код читаемым, легко тестируемым и позволяет избежать дублирования.

4. 💻 **Живой пример**:

```JavaScript
// Объявление функции
function calculateDiscount(price, discountPercent) {
  const discountAmount = (price * discountPercent) / 100;
  return price - discountAmount;
}

// Использование функции (вызов)
const finalPrice1 = calculateDiscount(1000, 10);
const finalPrice2 = calculateDiscount(5000, 20);

console.log(finalPrice1); // 900
console.log(finalPrice2); // 4000
```

5. ⚠️ **Типичные ошибки**:
- <u>Забывать инструкцию return</u>: если функция ничего не возвращает, она вернет `undefined`, и ты не сможешь использовать результат вычисления дальше.
- <u>Использование переменных из функции вне её</u>: переменные, созданные внутри функции (через `let`, `const`), <u>невидимы</u> за её пределами. Это называется *областью видимости*.
- <u>Неправильный порядок аргументов</u>: JavaScript передает их строго по очереди, если перепутать их местами — результат будет непредсказуемым.

5. 🛠 **Практический челлендж**:
Напиши функцию `greetUser(name`), которая принимает имя пользователя и возвращает строку `"Привет, [имя]!"`. Затем вызови эту функцию три раза с разными именами и выведи результат в консоль. <u>Подсказка</u>: используй оператор `+` для соединения строк или шаблонные строки с обратными кавычками.

✅ **Ключевой вывод**: Функции превращают разрозненные строки кода в <u>организованную систему</u>. Они позволяют один раз описать сложную логику и использовать её многократно, передавая разные данные через параметры. Теперь наш код стал модульным и профессиональным.
