1. 📌 **Тема**: Модули в JavaScript (ES Modules): Организация и архитектура кода

2. 🔍 **Разбор концепции**: 
Когда проект вырастает из одного файла в сотни, держать всё в одной куче становится невозможно. *Модульность* — это способ разбиения программы на независимые блоки, каждый из которых отвечает за свою область ответственности. В современном JavaScript мы используем стандарт *ES Modules* (ESM), который позволяет экспортировать (публиковать) функции, классы или переменные из одного файла и импортировать (подключать) их в другом. Это превращает ваш код в систему из отдельных *кирпичиков*, которые легко тестировать и поддерживать.

3. 💡 **Зачем это нужно?**: 
В реальной разработке вы никогда не будете писать весь код в одном файле. Представьте интернет-магазин: логика корзины лежит в `cart.js`, работа с API — в `api.js`, а утилиты для форматирования цен — в `utils.js`. Благодаря модулям, основной файл `index.js` просто собирает эти части воедино, не загромождая пространство имен и упрощая командную работу.

4. 💻 **Живой пример**:

```JavaScript
// mathUtils.js — файл, который экспортирует функционал
export const PI = 3.14;

export function add(a, b) {
  return a + b;
}

export default function multiply(a, b) {
  return a * b;
}

// main.js — файл, который потребляет функционал
import multiply, { PI, add } from './mathUtils.js';

console.log(add(5, 5)); // 10
console.log(multiply(2, PI)); // 6.28
```

5. ⚠️ **Типичные ошибки**:
- <u>Забывать расширение файла</u>: в отличие от Node.js без настроек или бандлеров, в браузерном ES-модуле импорт должен содержать полное имя файла (включая `.js`).
- <u>Конфликты имен</u>: попытка сделать два `export default` в одном файле. <u>Помните: в одном файле может быть только один дефолтный экспорт</u>, а именованных — сколько угодно.
- <u>Порядок выполнения</u>: многие забывают, что импортируемый код выполняется <u>автоматически</u> при первом импорте, даже если вы не вызываете функции из него.

6. 🛠 **Практический челлендж**:
Создайте два файла: `user.js` и `app.js`. 
- В `user.js` создайте объект `user` с полями `name` и `age` и экспортируйте его. 
- Также создайте там функцию `greet`, которая возвращает строку `"Привет, [имя]!"`. 
- В `app.js` импортируйте объект и функцию. 
- Выведите в консоль результат работы функции `greet`, передав в неё имя из объекта `user`.

✅ **Ключевой вывод**: Модули позволяют превратить хаос из функций и данных в структурированную систему. Использование `import` и `export` делает ваш код профессиональным, читаемым и удобным для масштабирования.
