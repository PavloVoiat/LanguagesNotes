1. 📌 **Тема**: Работа с сетевыми запросами: Fetch API и взаимодействие с сервером.

2. 🔍 **Разбор концепции**: 
Теперь, когда мы умеем управлять элементами на странице, пора научить их "общаться" с внешним миром. В современных веб-приложениях данные редко живут только внутри браузера — они приходят с сервера. *Fetch API* — это современный интерфейс, позволяющий выполнять сетевые запросы для получения или отправки данных. Под капотом он использует *Promise*, что делает работу с асинхронными сетевыми вызовами очень удобной. Мы отправляем *HTTP-запрос* (например, GET для получения данных), получаем *Response* (ответ), который нужно распарсить в нужный формат (обычно *JSON*).

3. 💡 **Зачем это нужно?**: 
Представь, что ты делаешь приложение "Погода". Тебе нужно получить данные о температуре в городе из внешнего сервиса. Ты отправляешь запрос на сервер, получаешь ответ в виде объекта, превращаешь его в JavaScript-структуру и выводишь на экран через DOM. Без этого современные сайты были бы статичными "картинками".

4. 💻 **Живой пример**:

```JavaScript
async function fetchUserData(userId) {
  try {
    const response = await fetch(`https://jsonplaceholder.typicode.com/users/${userId}`);
    
    // Проверяем, успешен ли статус ответа (200-299)
    if (!response.ok) {
      throw new Error('Ошибка сети');
    }

    const data = await response.json(); // Преобразуем тело ответа в объект
    console.log('Данные пользователя:', data);
  } catch (error) {
    console.error('Что-то пошло не так:', error.message);
  }
}

fetchUserData(1);
```

5. ⚠️ **Типичные ошибки**:
- <u>Забывать обрабатывать статус ответа</u>: метод `fetch` не выбрасывает ошибку, если сервер ответил 404 или 500. Всегда проверяй свойство `response.ok`.
- <u>Игнорировать асинхронность</u>: попытка обратиться к результату `fetch` без `await` или `.then()` приведет к получению `Promise` вместо реальных данных.
- <u>Отсутствие блока try/catch</u>: при потере интернет-соединения ваш код может "упасть", если вы не обернули запрос в конструкцию для обработки исключений.

6. 🛠 **Практический челлендж**:
Создай простую функцию, которая делает GET-запрос к публичному API (например, `https://jsonplaceholder.typicode.com/posts/1`) и выводит заголовок поста (`title`) в консоль. После этого добавь условие: если при запросе возникла любая ошибка, выведи в консоль сообщение `"Данные недоступны"`.

✅ **Ключевой вывод**: Сетевые запросы через `fetch` — это мост между твоим фронтендом и данными на сервере. Помни про <u>обязательную проверку response.ok</u> и правильную обработку ошибок через `try/catch`, чтобы твое приложение было надежным.
